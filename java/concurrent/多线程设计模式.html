# 单例模式

单例模式在并发情况下，需要保证线程安全，来实现单例对象的线程安全。

单例设计模式笔记 (opens new window)

在单例模式保证线程安全的几种方式中，静态内部类和枚举的效率最高，因为没有涉及到加锁等消耗性能的问题。

  • 静态内部类

    	/**
     * 静态内部类，在外部类被加载的时候不会立即加载。
     * 只有在调用该类里的参数或方法时，对象实例才会被加载。达到了懒加载的效果。
     * 在类加载的时候，是线程安全的。达到了线程安全的效果。
     */
    private static class SingletonStaticClass {
        private static final SingletonStatic SINGLETONSTATIC = new SingletonStatic();
    }

    public static SingletonStatic getInstance() {
        return SingletonStaticClass.SINGLETONSTATIC;
    }

# 不变模式

在并发情况下，若对象在创建后，内部状态永久不会被改变，也能保证并发情况下的线程安全问题。依靠对象的不可变性，可以确保并发情况下访问到的对象是相同的，这就是不变模式。

不变模式情况下，不需要使用同步操作来实现线程安全。

# 不变模式实现

在 Java 中，实现不变模式，确保对象在创建后不会发生任何改变。

  • 去除对象的 setter 方法以及修改属性的方法。

  • 将所有属性设置为私有，用 final 标记。

  • 确保没有子类可以重载修改对象行为。

  • 有一个创建完整对象的构造函数。

    public final class Product {

    private final String no;
    private final String name;
    private final double price;

    public Product(String no,String name,double price){
        super();
        this.no=no;
        this.name=name;
        this.price=price;
    }

    public String getNo() {
        return no;
    }

    public String getName() {
        return name;
    }

    public double getPrice() {
        return price;
    }
}

# 不变模式的应用

在 JDK 中，不变模式的应用很广泛。最经典的就是 java.lang.String 类。

所有的包装类都是使用不变模式实现的。
