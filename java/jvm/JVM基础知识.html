# JVM学习笔记

# 基础

# JVM、JDK、JRE的区别

  • JVM 是 Java虚拟机，是实现 Java 跨平台的核心部分。JVM 隐藏了和操作系统交互的细节，使得被编译后的 Java程序，能在虚拟机运行，并且不与操作系统直接交互。

    这也是一次编译，能到处运行的原因。只要编译后，使用编译后的 .class 文件借助 JVM 虚拟机可以运行到到多种平台。

  • Jre 是 Java 的运行环境，Jre 包含了 Java虚拟机 JVM，并且还包含 JVM 执行 .class 文件时所需要的基础类库。

    对于普通用户来说，使用 Jre 就可以运行 Java 程序。

    • bin 目录对应就是 JVM，而 lib 目录对应 JVM 需要的基础类库。

    • Jre 的 lib 目录下包含了 rt.jar。

      • rt.jar：Java基础类库，也就是Java doc里面看到的所有的类的class文件。

        Java 中常使用到的 lang 包、util 包都在这个 jar 中。

  • JDK 是面向开发人员的，在包含 JRE 的基础上，还包含了编译工具。

    其中 Java 工具（jstack、jps）等由 Jdk 提供。

    • JDK 目录下包含着 Jre，还包含 lib 目录，

    • JDK 的 bin 目录包含了一些 Java 工具的可执行文件。

    • JDK 的 lib 目录包含了 tools.jar 和 dt.jar。

      • tools.jar：是系统用来编译一个类的时候用到的，即执行 javac 的时候用到。
      • dt.jar：dt.jar 是关于运行环境的类库，主要是swing包。

# 线程运行诊断

# cpu 占用过多

  1. top 命令分析某个进程的 cpu 占用情况

  2. ps 命令分析该进程占用 cpu 多的线程。

    ps H -eo pid,tid,%cpu |grep 32655
    1

  1. 使用 jstack 分析 cpu 使用量高的线程，pid = 32665。

    jstack 命令产生的日志里 pid 是 16进制，所以需要转换 pid 为 16 进制 7F99 。

    可以看到定位到了线程，并且定位到了线程的代码位置。

# 排查死锁

  1. 使用 jstack 命令排查死锁。

    jstack PID
    1

  2. 使用 jconsole 检测死锁。

JDK 提供的可视化工具。
